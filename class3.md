## Version Control

Stores changes made to files via versions.

> Features:
>
>> . File Reversion
>> 
>> . Record of Changes
>> 
>> . Change Comparisons

## Types:

- **Local Version Control**
  - Changes stored via database on hard disk
  
- **Centralized Version Control**
  - Changes saved via a server that can be accessed by multiple users
  - Disadvantage: Server is a single point of failure
  
> Freatures:
> 
>> - Enables collaboration betweeen programmers
>> 
>> - Allows for different level of revision privileges

- **Distributive Version Control**
  - Users repositories mirror data for backup and can be used to update info lost via server
